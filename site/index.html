Why V1 did not hold in production

V1 had energy, but it had drift. Work moved fast, then context got lost, dispatch got noisy, and the desktop launch story became confusing. The core issue was not one bug; it was system shape.

1. Dispatch noise

Too many queued or stale requests made signal quality weak and operators had to guess what still mattered.

2. Recency drift

Health could look degraded only because timestamps were stale, even after a fresh pulse in another root.

3. Dual-root confusion

Repo root and AppSupport root were not always read as one operating picture, so operators saw mixed truth.

4. Launch channel confusion

Dev Live applet behavior made the primary icon experience unclear for non technical users.

5. Provider dependency risk

Antigravity credit/outage windows could stall lanes without a strict outage policy.

Why V2 works now (fix map)

V2 uses explicit contracts. Every high-risk pain point now has an owner, a deterministic gate, and a check command.

Role of add-ons in Cockpit V2

Add-ons are not decoration. They are controlled extension layers. We split them in two lanes so operators can reason about risk quickly.

Runtime add-ons

"Can the app run and ship clean?"

  • PySide6 Addons/Essentials for UI runtime modules.
  • Packaging scripts for macOS release builds.
  • Launch channel scripts (release vs dev live).
  • Healthcheck and pulse command wrappers.

Runtime add-ons are judged by stability, startup behavior, and rollback safety.

Product add-ons

"Can agents do useful work with guardrails?"

  • Skills catalog and installer governance.
  • Mission templates and dispatch packets.
  • Vulgarisation reports and operator brief cards.
  • Policy gates for credits, recency, and approval.

Product add-ons are judged by evidence quality and decision clarity.

FAQ

Why is @leo paused right now?

Current policy (Wave16) keeps AG-targeted lanes paused while Antigravity credits are unavailable. This protects reserve floor and keeps control on core backend/advisory delivery.

What is the difference between release and dev live?

Release mode is the single-icon packaged app target for operators. Dev live remains available for engineering iteration but is no longer the primary launch icon path.

Do add-ons reduce safety?

Not by default. Add-ons are split by lane (runtime/product), documented, and gated with tests, receipts, and owner accountability.

How do we know V2 status is fresh?

We publish runbook cadence plus gate checks for both roots and keep run receipts with timestamps and command traces.
