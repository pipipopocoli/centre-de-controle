1. Contexte & Justification

1.1 Les 5 Murs de la V1

# Mur Preuve dans le code Impact
1 Agents hardcodés auto_mode.py:296 — _agent_platform() = if/else statique 🔴 Impossible d'ajouter un agent sans modifier Python
2 Pas de mémoire cross-projet brainfs.py:63 — load_profile() lit UN profil par projet 🔴 Amnésie entre projets
3 Skills décoratives brainfs.py:19 — load_skills() → jamais injecté dans le prompt 🟡 Les compétences ne changent rien
4 Dispatch fragile auto_mode.py:458 — 1 requête à la fois, synchrone 🔴 Requête perdue si crash
5 Single Runtime auto_mode.py:296 — 5/7 agents sur Codex 🔴 80% bloqué si Codex down

2. Les 5 Plugins

PHASE 1 Plugin 1 : Agent Registry & Hiérarchie

Pourquoi : On ne peut pas ajouter un agent sans modifier le code Python. _agent_platform() est un if/else hardcodé.

Solution : Un fichier brainfs/agents_registry.json — source de vérité unique pour tous les agents.

  "agents": {
    "clems":  { "level": 0, "role": "Orchestrator",    "platform": "antigravity", "manages": ["victor","leo"] },

PHASE 1 Plugin 2 : SOUL.md & Skills Dynamiques

Pourquoi : Les agents n'ont pas de personnalité. skills.json n'est jamais injecté dans les prompts.

Solution :

  • SOUL.md = fiche d'identité de chaque agent (ton, règles, mémoire)
  • Skill Library = brainfs/library/ avec des dossiers par compétence contenant SKILL.md

Flux d'injection :

sequenceDiagram participant AM as Auto-Mode participant REG as Agent Registry participant SOUL as brainfs/souls/ participant LIB as brainfs/library/ participant RT as Runtime (Codex) AM->>REG: Quel agent pour cette tâche ?
