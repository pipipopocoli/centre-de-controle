1. La Proposition VICTOR (Architecture & Risque)

1.1 Le Mur de la Peur (Risk Wall)

Problème : On construit trop vite, on casse tout, et on ne sait pas revenir en arrière.

Mur Preuve dans le code Impact
Pas de Backup session_manager.py — écrase ui_session.json à chaque write 🔴 Corruption = Perte totale de l'état
Pas de Rollback git — commits "WIP" mélangés 🔴 Impossible de revert une feature sans tout casser

1.2 Solution : "Implementation Lots"

Plugin Action Valeur
VICTOR Session Snapshots Backup ui_session.json.bak avant écriture 🛡️ Résilience (Crash-proof)
VICTOR Severity Matrix Tagger chaque issue (Sev1, Sev2, Sev3) ⚖️ Priorisation objective

2. La Proposition AGENT-11 (Faisabilité & Scope)

2.1 Le Mur de la Complexité

Problème : On veut tout faire (Vector DB, LLM local, Voice Mode) et on ne finit rien.

Mur Preuve Impact
Feature Bloat ROADMAP.md — 35 items "High Priority" 🔴 Rien ne sera livré à 100%
Scope Creep Demandes utilisateur "ah et aussi ça..." 🔴 Épuisement de l'équipe (ou des tokens)

2.2 Solution : "The Pruning Shears"

A11 Strict 12-Item List Hardcap (MAX_WIP=12) dans task.md 🎯 Focus absolu
A11 Anti-Complexity Gate Rejet automatique si > 3 fichiers touchés ✂️ Simplicité forcée

3. La Proposition AGENT-4 (Contrôle & Déterminisme)

3.1 Le Mur de l'Entropie

Problème : Les transitions d'état sont floues. "Done" veut dire "Coded" pour l'un, "Deployed" pour l'autre.

Mur Preuve Impact
Flou Artistique task.md — [x] coché sans lien vers la preuve 🔴 Faux positifs ("C'est fini" -> Bug en prod)
Drift Fichiers modifiés sans ticket associé 🔴 Code orphelin

3.2 Solution : "The Token Matrix"

Plugin Action Valeur
A4 Evidence Links Obligation de [Ref](file://...) pour tout [x] 🔗 Traçabilité totale
A4 Gate Scripts verify_X.sh pour chaque feature 🤖 Validation binaire (Pass/Fail)
