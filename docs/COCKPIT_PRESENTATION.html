Humans

3) Automation pipeline (the core loop)

A mention in chat becomes a run request. Auto-mode routes it to an agent inbox. Auto-send (optional) executes it by pasting into the right external thread.

(queue)
→

→
runs/inbox/<agent>.ndjson
AUTO-SEND OFF
(dispatch only)

AUTO-SEND ON
(open app + paste + enter)
→
Agent replies via MCP
→
Lifecycle close
(reply_received)

Why "apps pop but nothing runs" happens

  • If AUTO-SEND is OFF: we queue/dispatch only. No keystrokes are sent.
  • If macOS blocks keystrokes: Accessibility permissions are missing.
  • If thread naming drifts: window matching can skip sends (safety guardrail).

What is solid in the plan

  • Non-destructive: never auto-overwrite memory.md.
  • Determinism + caps: keeps output stable and small.
  • Scope lock: cockpit only (no cross-project retrieval).
  • Storage is explicit and auditable (files you can open).

What we corrected / want clarified

  • Paths should be written in terms of <projects_root>/<project_id> (runtime is App Support by default).
  • Compaction caps should be explicitly enforced (lines, per-section items, truncation).
  • Index schema should be minimal but stable (facts/decisions/open_loops/links).

Suggested next step for memory (after gate)

  • Implement memory.index.json generator (per agent).
  • Wire compaction output to a one-click UI action (still non-destructive).
  • Add 1-2 tests: determinism and caps.

6) Operator workflow (60 seconds)

  1. Open Cockpit app, select project {{PROJECT_ID}}.
  2. Ensure AUTO-MODE = ON.
  3. If you want real execution: AUTO-SEND = ON.
  4. Click Preflight, then Run once.
  5. In chat, write a mission: @victor ... or @leo ... or @agent-1 ...
  6. Watch: Pending goes down after replies; cards update; requests close with reason reply_received.
