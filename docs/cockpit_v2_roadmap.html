1. État Actuel & Architecture ("As-Is")

Le projet Cockpit est actuellement en fin de phase "Tournoi". L'architecture repose sur un modèle Agent-As-Code simple mais rigide.

💡 Analogie : Le Prototype de Course Le Cockpit actuel est comme un prototype de Formule 1 assemblé dans un garage.
  • 🏎️ Moteur (Codex) : Très puissant.
  • 🌪️ Vitesse (Auto Mode) : Très rapide.
  • ⚠️ Sécurité : Aucune. Pas d'airbag (Atomic Write), pas de freins (Cap), et le tableau de bord (Bible) est illisible.

Les 5 Murs (Dettes Techniques)

Problème Preuve Code Impact
1. Agents Hardcodés auto_mode.py (if/else statique) Mise à jour impossible sans dev Python.
2. Amnésie Totale Pas de mémoire cross-projet L'IA répète les mêmes erreurs.
3. Skills Fantômes load_skills() inutilisé Les experts n'ont aucune expertise.
4. Dispatch Fragile Appels synchrones Crash app = Perte de commande.
5. Single Runtime Dépendance Codex 100% Si Codex tombe, tout s'arrête.

2. Objectif du Tournoi ("The Big Merge")

Le tournoi n'avait pas pour but de désigner un vainqueur, mais de trouver les pièces manquantes pour passer du Prototype à la Série.

Les agents ont proposé des solutions radicales :

  • 👮 Agent-11 : Contrôle strict (Gatekeeping).
  • 🧠 Agent-4 : Profondeur d'analyse (Risks/Skills).
  • 🎨 Leo : Clarté de l'information (Bible UI).

5. Phase 2 : Core Expansion (Les 5 Plugins V2)

Une fois le socle stabilisé par le "Big Merge", nous intégrons les 5 Plugins de la Spécification Technique V2 pour transformer la plateforme.

PLUGIN 1 Agent Registry & Hiérarchie

Solution : Externaliser la définition des agents dans brainfs/agents_registry.json.

"victor": { "role": "Backend Lead", "platform": "codex", "rank": "L1" }

PLUGIN 2 Mémoire & Skills Dynamiques

Solution : Injection contextuelle des compétences.

sequenceDiagram App->>Registry: Qui pour cette tâche ? Registry-->>App: Victor App->>Skills: Charger 'python-expert' Skills-->>App: Instructions Python App->>LLM: Prompt = Context + Skills + Tâche

PLUGIN 3 Task Manager UI

Solution : Un tableau de bord "Aéroport" pour voir en temps réel qui fait quoi (Clems: En mission, Victor: En attente).

PLUGIN 4 Platform Router (Multi-Runtime)

Solution : Briser la dépendance à Codex. Le routeur choisit le meilleur moteur.

  • 🟢 Tâche simple -> Ollama (Local) (Gratuit, Rapide)
  • 🟡 Tâche complexe -> Antigravity (Puissant)
  • 🔴 Tâche créative -> Codex/Gemini (Avancé)

PLUGIN 5 Mémoire Progressive

Solution : Indexation automatique (SQLite FTS5 puis ChromaDB) pour permettre aux agents de se "souvenir" des projets passés.

6. Questions & Zones Inexplorées

❓ La Question du Coût Plus on ajoute d'intelligence (Gatekeeper vérifie, Mémoire cherche, Skills injectent), plus chaque action coûte cher en tokens.

Piste : Introduire un "Budget Token" par étape. Si une tâche est triviale, on interdit l'accès à la Mémoire Long-Terme.

L'Humain dans la boucle ?

Avec le Plugin 3 (Task Manager), l'objectif est de redonner le contrôle à l'opérateur. L'Auto Mode ne doit pas être une boîte noire, mais une "usine de verre".
